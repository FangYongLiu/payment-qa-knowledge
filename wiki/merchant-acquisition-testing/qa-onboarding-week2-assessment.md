# QA新人第二周入职评估清单

本页用于验证新QA成员是否掌握标准QA工作流，并能在导师轻度指导下，独立参与从用例设计到生产发布的完整交付生命周期。

## 评估目的

- 验证新成员对团队 QA 流程的理解
- 验证其能否参与完整交付生命周期：从测试用例设计到生产发布

## 评估说明

- 任务应在最少指导下完成
- 所有工作必须记录在相应工具中（JIRA / Zephyr）
- 需提供证据：链接、截图、工单 ID 等
- 由导师审阅并给予反馈

## Task 1 — Zephyr 测试用例编写

**目标**：展示设计并撰写测试用例的能力。

**要求**：
- 在 Zephyr 中为指定功能创建测试用例
- 遵循团队的用例模板与标准
- 包含清晰的 preconditions、steps、expected results、test data
- 将测试用例关联至对应的 JIRA story/task

**完成记录字段**：
- Zephyr Test Cycle
- Test Case IDs / Links
- Reviewer Comments (if any)

## Task 2 — 缺陷上报与跟踪

**目标**：展示提交高质量缺陷单的能力。

**要求**：
- 执行测试用例并识别至少一个问题（真实或模拟）
- 在 JIRA 中按团队 bug 模板提交缺陷
- 包含 reproduction steps、expected vs actual result、environment、logs/screenshots
- 跟踪 bug 状态在工作流中的流转

**完成记录字段**：
- Bug Ticket ID
- Severity / Priority
- Final Status
- Evidence

## Task 3 — JIRA 工单生命周期参与

**目标**：理解工单生命周期与状态流转。

**要求**：
- 参与 JIRA task/story 的完整生命周期，包含：
  - Requirement review
  - Test preparation
  - Execution
  - Verification
- 按团队工作流恰当更新工单状态
- 添加有意义的评论以记录测试进展

**完成记录字段**：
- JIRA Ticket ID
- Lifecycle Participation Summary

## Task 4 — 端到端发布流程参与

**目标**：体验完整的部署与发布过程。

**要求**：在导师指导下参与端到端交付流程：
- Code deployment to test environment
- Test execution and result reporting
- Verification of fixes（如适用）
- Release preparation
- Production deployment 的观察或参与

**完成记录字段**：
- Feature / Release Name
- Environment(s) Tested
- Key Activities Performed
- Issues Found (if any)
