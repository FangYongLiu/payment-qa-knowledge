# WPS Payroll QA 入职考核 - Week 2

本页用于验证新入职 QA 是否掌握标准 QA 工作流，并能独立参与从用例设计到生产发布的完整交付生命周期。

## 考核目的

- 验证新人对标准 QA 工作流的理解
- 验证新人具备参与完整交付生命周期的能力（从测试用例设计到生产发布）

## 考核说明

- 在最少指导下独立完成任务
- 所有工作需记录在对应工具中（JIRA / Zephyr）
- 提供证据（链接、截图、ticket ID）
- 由 Mentor 审核并给出反馈

## Task 1 — Zephyr 测试用例编写

**Objective**：展示设计与编写测试用例的能力。

**Requirements**：
- 在 Zephyr 中为指定功能创建测试用例
- 遵循团队的测试用例模板与标准
- 用例需包含清晰的 preconditions、steps、expected results、test data
- 将测试用例关联到对应的 JIRA story/task

**Record of Completion**：
- Zephyr Test Cycle
- Test Case IDs / Links
- Reviewer Comments (if any)

## Task 2 — Bug 报告与跟踪

**Objective**：展示高质量缺陷报告能力。

**Requirements**：
- 执行测试用例并识别至少一个 issue（真实或模拟）
- 按团队 bug 模板在 JIRA 提交缺陷
- 包含 reproduction steps、expected vs actual results、environment、logs/screenshots
- 跟踪 bug 在工作流中的状态变化

**Record of Completion**：
- Bug Ticket ID
- Severity / Priority
- Final Status
- Evidence

## Task 3 — JIRA 工作流参与

**Objective**：理解 ticket 的生命周期与状态流转。

**Requirements**：
- 参与 JIRA task/story 的完整生命周期，包括：
  - Requirement review
  - Test preparation
  - Execution
  - Verification
- 按团队工作流更新 ticket 状态
- 添加有意义的 comments，记录测试进展

**Record of Completion**：
- JIRA Ticket ID
- Lifecycle Participation Summary

## Task 4 — 端到端发布流程参与

**Objective**：体验完整的部署与发布流程。

**Requirements**（在 Mentor 指导下参与端到端交付流程）：
- Code deployment to test environment
- Test execution and result reporting
- Verification of fixes（如适用）
- Release preparation
- Production deployment observation or participation

**Record of Completion**：
- Feature / Release Name
- Environment(s) Tested
- Key Activities Performed
- Issues Found (if any)

## 自评（New QA Member 填写）

- 简要总结对 QA 流程与发布工作流的理解

## Mentor 评估

**Overall Result**：
- Pass
- High Risk
- Fail

**Mentor Comments**：由 Mentor 填写反馈意见。
