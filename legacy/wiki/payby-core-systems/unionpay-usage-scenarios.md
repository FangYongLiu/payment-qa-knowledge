# UnionPay应用场景说明

本页说明 UnionPay 在客户端的三类使用入口（Money 页面、付款码、扫商户码）及其在不同用户状态下的行为，并补充外币支付的特殊处理。开卡与激活的流程细节见 [[unionpay-card-issuance-activation]]。

## 通用前置状态

不同入口对用户状态的处理逻辑相似，统一遵循：

- **未 KYC**：拒绝或引导去 KYC。
- **已 KYC 未激活**：引导进入激活页面，激活完成后回到原入口。
- **已激活**：进入正常功能。

激活完成后才能使用 UnionPay 能力，端到端流程见 [[flow_unionpay_card_activation]]；开卡端到端流程见 [[flow_unionpay_card_issuance]]。

## Money 页面入口

- **未 KYC 用户**：点击入口直接报错。
- **已 KYC 未激活**：引导进入激活页面，激活成功后跳转至 UnionPay 页面。
- **UnionPay 页面功能**：
  - 设置 / 修改 PIN
  - 查看银联卡信息
  - 安卓手机：提供开通 NFC 按钮；在支持银联的 POS 机刷卡时，由银联调用 `upic` 扣款接口，从用户钱包扣款。

## 付款码入口

- **未 KYC 用户**：付款码页面仅展示自有付款码。
- **已 KYC 用户**：付款码页面提供 Tab，可在自有付款码与银联付款码之间切换。
  - **未激活**：银联付款码不展示，显示"激活"按钮，点击进入激活流程；激活成功后回到付款码页面展示银联付款码。
  - **已激活**：直接展示银联付款码，商户扫码即可完成支付，由银联调用 `upic` 扣款接口。

## 扫商户码入口

- **未 KYC 用户**：扫码后引导去做 KYC。
- **已 KYC 未激活用户**：扫码后进入激活流程。
- **已激活用户**：扫码进入下单页面（我方页面），页面包含：
  - 商户信息
  - 金额输入框
  - `Pay` 按钮

### 外币场景特殊处理

扫商户码场景下若涉及外币，处理方式如下：

- 输入框展示**外币金额**，并显示"约等于 AED 金额"。
- 汇率来自数据库，**每日从银联官网同步**。
- 点击 `Pay` 呼起收银台，金额以 AED **预估值**展示。
- 提交支付时，银联返回**真实的 AED 金额**，可能与收银台预估不同。
- 用户端流程在提交后即结束，因此可能出现**实际扣款金额与收银台展示金额存在差异**。

## 注意事项

- 开卡过程对用户无感知，激活是用户可见流程，必须合规执行。
- 外币支付需特别注意预估金额与实际扣款金额可能存在偏差。
- `upic` 扣款接口是所有 UnionPay 支付场景的资金落地关键。
